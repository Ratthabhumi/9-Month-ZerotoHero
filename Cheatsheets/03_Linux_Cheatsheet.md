# 🐧 Linux Fundamentals Cheatsheet (Week 2 Day 3)

ในฐานะ DevOps Engineer การใช้งาน Command Line ใน Linux คือทักษะพื้นฐานที่สำคัญที่สุด!

## 1. การจัดการไฟล์และโฟลเดอร์ (File & Directory Management)
- `pwd` : ดูว่าตอนนี้เราอยู่โฟลเดอร์ไหน (Print Working Directory)
- `ls` : ดูรายชื่อไฟล์และโฟลเดอร์ (`ls -l` ดูรายละเอียด, `ls -a` ดูไฟล์ซ่อน)
- `cd <path>` : เปลี่ยนโฟลเดอร์ (Change Directory)
  - `cd ..` ย้อนกลับ 1 ขั้น
  - `cd ~` กลับไปที่ Home ของ User
- `mkdir <name>` : สร้างโฟลเดอร์ใหม่
- `touch <file>` : สร้างไฟล์เปล่าๆ
- `rm <file>` : ลบไฟล์ (`rm -r <folder>` ลบโฟลเดอร์และของข้างใน)
- `cp <source> <dest>` : ก๊อปปี้ไฟล์
- `mv <source> <dest>` : ย้ายไฟล์ หรือ เปลี่ยนชื่อไฟล์

## 2. การค้นหาและดูข้อมูลในไฟล์ (Search & View)
- `cat <file>` : อ่านข้อมูลในไฟล์ทั้งหมดออกมาบนหน้าจอ
- `head <file>` : ดูข้อมูล 10 บรรทัดแรก (`head -n 5` ดู 5 บรรทัดแรก)
- `tail <file>` : ดูข้อมูล 10 บรรทัดสุดท้าย (ใช้ `tail -f` เพื่อดู Log แบบ Real-time)
- `grep "คำที่ต้องการ" <file>` : ค้นหาข้อความในไฟล์ (สำคัญมากตอนหา Error ใน Log)
- `find <path> -name "ชื่อไฟล์"` : ค้นหาไฟล์ในโฟลเดอร์

## 3. Pipes (`|`) และ Redirection (`>`)
ใช้เวทมนตร์ส่งต่อข้อมูลจากคำสั่งนึงไปอีกคำสั่งนึง
- `>` : นำผลลัพธ์ไปเขียนทับลงไฟล์ เช่น `echo "Hello" > test.txt`
- `>>` : นำผลลัพธ์ไปต่อท้ายไฟล์ (Append) เช่น `echo "World" >> test.txt`
- `|` (Pipe) : เอาผลลัพธ์ของคำสั่งซ้าย ส่งเป็น Input ให้คำสั่งขวา 
  - *ตัวอย่าง:* `cat server.log | grep "ERROR"` (อ่านไฟล์ แล้วกรองเอาเฉพาะบรรทัดที่มีคำว่า ERROR)

## 4. File Permissions (สิทธิ์การจัดการไฟล์)
Linux ให้ความสำคัญกับความปลอดภัยมาก ทุกไฟล์จะถูกกำหนดสิทธิ์ไว้เสมอ
- `r` (Read) = 4 : สิทธิ์อ่าน
- `w` (Write) = 2 : สิทธิ์เขียน/แก้ไข
- `x` (Execute) = 1 : สิทธิ์สั่งรัน (สำหรับ Script)

การเปลี่ยนสิทธิ์ใช้คำสั่ง `chmod`
- `chmod +x script.sh` : ให้สิทธิ์รันไฟล์นี้ได้
- `chmod 755 script.sh` : เจ้าของอ่านเขียนรันได้ (4+2+1=7), คนอื่นอ่านและรันได้อย่างเดียว (4+1=5)
