# 08_Networking_PortMapping_Cheatsheet.md (Week 3)

## 🏢 Concept ของ IP Address & Port Number

### 1. การเปรียบเทียบในชีวิตประจำวัน (Analogy)
- **IP Address:** ที่อยู่หลักของเครื่องเซิร์ฟเวอร์ เปรียบเหมือน **"ที่อยู่คอนโด / บ้านเลขที่"** ช่วยให้ส่งพัสดุมาถูกอาคาร
- **Port Number:** เลขช่องทางของแต่ละโปรแกรม เปรียบเหมือน **"เลขห้องพักในคอนโด"** ช่วยให้รู้ว่าต้องส่งพัสดุไปให้โปรแกรมไหน
  - มีค่าตั้งแต่ `0` ถึง `65,535`
- **Port Conflict (พอร์ตชนกัน):** ปัญหาที่เกิดขึ้นเมื่อ 2 โปรแกรมพยายามเปิดใช้ Port เดียวกันพร้อมกันในเครื่องเดียวกัน ระบบจะไม่ยินยอมเพราะไม่สามารถตัดสินใจได้ว่าจะส่งข้อมูลไปให้โปรแกรมไหน (เหมือนไม่รู้ว่าจะส่งพัสดุไปห้องไหน)

## 📌 Well-Known Ports (พอร์ตมาตรฐานที่ต้องจำ)

| Port | Protocol / Service | หน้าที่ / การใช้งาน |
|---|---|---|
| `22` | **SSH** | รีโมทเข้าควบคุมเซิร์ฟเวอร์ Linux ผ่าน Terminal |
| `80` | **HTTP** | รับส่งข้อมูลเว็บแบบธรรมดา (ไม่มีการเข้ารหัส) |
| `443` | **HTTPS** | รับส่งข้อมูลเว็บแบบปลอดภัย (เข้ารหัสผ่าน SSL/TLS) |
| `3306` | **MySQL** | การเชื่อมต่อฐานข้อมูล MySQL / MariaDB |
| `5432` | **PostgreSQL** | การเชื่อมต่อฐานข้อมูล PostgreSQL |
| `6379` | **Redis** | In-memory Cache / Database |

## ☁️ ความเชื่อมโยงระหว่าง Cloud, Container และ Port Mapping

1. **Cloud (เช่น AWS EC2):** เปรียบเหมือน **"ที่ดินเปล่า / รั้วบ้านใหญ่"** ที่มีประตูทางเข้าหลัก
2. **Container (เช่น Docker):** เปรียบเหมือน **"ตู้สินค้า / ห้องทำงานส่วนตัว"** ที่แพ็คโค้ดและสภาพแวดล้อมไว้ครบถ้วน รันแยกเน็ตเวิร์กอยู่ข้างใน (IP ภายใน)
3. **Port Mapping (เช่น `-p 8080:80`):** คือ **"การเจาะท่อส่งของ"** เชื่อมระหว่างหน้าบ้าน (Host Port 8080) กับห้องทำงานข้างใน (Container Port 80) ทำให้ผู้ใช้งานผ่านอินเทอร์เน็ตข้างนอก สามารถทะลุเข้ามาใช้งานแอปภายใน Container บน Cloud ได้ถูกจุด

---

## 🔀 หลักการทำงานของ Port Mapping (Host vs Guest)

**รูปแบบการตั้งค่าใน Docker / Container:**
```text
-p <Host_Port>:<Container_Port>
```

**ประโยชน์หลัก:**
1. **แก้ปัญหาพอร์ตชนกันเมื่อรันหลายแอป:** สามารถรัน Container 2 ตัวที่เปิด Port 80 เหมือนกันข้างในเครื่องเดียวกันได้ โดยการแยก Host Port ฝั่งนอก:
   - Web App 1: `-p 8080:80` (Host 8080 -> Container 1)
   - Web App 2: `-p 8081:80` (Host 8081 -> Container 2)
2. **ความปลอดภัย (Security & Isolation):** คนข้างนอกมองไม่เห็น IP หรือ Port จริงภายใน Container เข้าถึงได้เฉพาะ Port ที่เราทำ Mapping ออกมาเท่านั้น

## 🔍 คำสั่งตรวจสอบการเปิดใช้งาน Port (Port Inspection)

### 🪟 Windows (PowerShell)
```powershell
# ตรวจสอบว่าโปรเซสไหน (OwningProcess / PID) กำลังจอง Port 8000 อยู่
Get-NetTCPConnection -LocalPort 8000 | Select-Object LocalAddress, LocalPort, State, OwningProcess
```

### 🐧 Linux / macOS
```bash
# ดูรายการ Port ทั้งหมดที่กำลัง Listen อยู่ พร้อมชื่อโปรเซส (PID)
ss -tulpn | grep 8000

# หรือใช้คำสั่ง netstat แบบดั้งเดิม
netstat -tulpn | grep 8000
```
