# 06_Networking_DNS_Cheatsheet.md (Week 3)

## 🌐 DNS (Domain Name System) คืออะไร?
ระบบที่เปรียบเสมือน **"สมุดหน้าเหลืองของอินเทอร์เน็ต"** ทำหน้าที่แปลงชื่อโดเมนที่มนุษย์จำได้ง่าย (เช่น `google.com`) ให้กลายเป็น IP Address ที่คอมพิวเตอร์ใช้คุยกัน (เช่น `142.250.190.46`)

---

## 🗂️ DNS Records พื้นฐาน (ประเภทของข้อมูล)
- **A Record (Address):** ชี้ชื่อโดเมนไปหา **IP Address (IPv4)** โดยตรง
- **CNAME Record (Canonical Name):** ชี้ชื่อโดเมนไปหา **ชื่อโดเมนอื่น** (นามแฝง) ไม่ใช่ IP
- **MX Record (Mail Exchange):** ระบุว่า **Email Server** ของโดเมนนี้คือเครื่องไหน (ใช้สำหรับรับส่งอีเมล)
- **TXT Record (Text):** เก็บข้อความธรรมดา มักใช้สำหรับ **ยืนยันความเป็นเจ้าของโดเมน** (Domain Verification) หรือระบบป้องกันสแปม (SPF)

---

## 💻 คำสั่งตรวจสอบ DNS พื้นฐาน (Hands-on)
ใช้สำหรับตรวจสอบว่าโดเมนนั้นๆ ชี้ไปที่ไหน (มักใช้ตอนแก้ปัญหาเว็บล่ม หรือย้ายโฮสต์)

**1. หา A Record (ดู IP ของเว็บ):**
```bash
nslookup google.com
# หรือ
dig google.com
```

**2. หา MX Record (ดู Mail Server):**
```bash
nslookup -type=mx google.com
# หรือ
dig mx google.com
```

## ⏳ TTL (Time To Live) & ระบบ Cache
**TTL** คือ **"อายุไขของข้อมูล DNS"** มีหน่วยเป็นวินาที ระบุว่า DNS Resolver (เช่น ของค่ายเน็ต True, AIS) จะจำคำตอบ (Cache) ไว้นานแค่ไหนก่อนจะกลับไปถามเซิร์ฟเวอร์เจ้าของโดเมนใหม่

**💡 ทำไมเราถึงต้องมีระบบจำ (Cache) และไม่วิ่งไปถามเซิร์ฟเวอร์หลัก (Authoritative Server) โดยตรงทุกครั้ง?**

1. **ความเร็ว (Performance):** 
   การดึงข้อมูลจาก Cache ของเน็ตค่ายใกล้บ้าน ใช้เวลาเพียง `1-2 ms` แต่ถ้าวิ่งไปถามเซิร์ฟเวอร์หลักที่อาจจะอยู่ต่างประเทศ ต้องใช้เวลา `50-100+ ms` การมี Cache ทำให้คนทั้งโลกเข้าเว็บได้ไวขึ้นมหาศาล
   
2. **ลดภาระเซิร์ฟเวอร์ (Server Load & DDoS Prevention):** 
   หากคนทั้งโลกต้องดึงข้อมูลจากเซิร์ฟเวอร์หลักโดยตรงทุกครั้ง (ไม่มี Cache) เซิร์ฟเวอร์หลักจะทำงานหนักจนอาจถึงขั้นล่ม (Crash) การมีระบบ Cache จึงช่วยกระจายภาระ (Load Distribution) ไปยังผู้ให้บริการอินเทอร์เน็ตทั่วโลก

**⚠️ ข้อควรระวังสำหรับ DevOps:**
ก่อนจะทำการย้ายเซิร์ฟเวอร์ หรือเปลี่ยน IP ของเว็บไซต์ **ต้องลดค่า TTL ให้สั้นลง (เช่น 300 วินาที / 5 นาที) ล่วงหน้า** เพื่อให้ Cache ทั่วโลกหมดอายุไวขึ้น คนจะได้อัปเดตไปเจอ IP ใหม่ได้เร็วขึ้น
